## 一、产品定位

板鸭留子 Alive 是一个**留学生(主要面向西班牙)的本地优先生活操作系统**。
"板鸭"= 西班牙(谐音 España),"留子"= 留学生。

**用户特征**:
- 中文用户,生活在西班牙
- 多币种(EUR / USD / DKK / CHF / GBP / CNY)
- 重度使用记账、日历、日记、番茄钟
- 移动端 + 桌面端混合使用

**产品基调**:
- 温暖、有人情味,不冷冰冰
- 玻璃拟态(glassmorphism)是已确立的视觉方向
- 暖色调为默认(暖桃黄昏)+ 多主题切换支持
- "记忆生活"而不是"管理生活"——它陪伴感强,不是生产力工具

## 二、当前问题(用户原话整理)

### 整体感受
> "这个菜单太丑了"
> "字体太小了,数字的字体太丑了,而且太细"
> "整个交易记录乱乱的,我更喜欢每个部分对齐的形式"
> "番茄钟的弹窗黑框太丑了"

### 失去了什么(对比之前的 HTML 老版)
> "我 HTML 那些欢迎页、早晨的八段锦、完成的动画都给我删了??"

→ 用户怀念的是**有温度的小细节**:
- 启动欢迎页(spring 弹簧动画 + 渐变文字)
- 早晨打开 app 时的"八段锦"提示
- 完成番茄钟 / 完成记账后的庆祝动画

### 具体的视觉/排版痛点
1. 顶部 nav bar 把所有菜单平铺显得拥挤
2. 交易记录每行的元素没对齐(图标/标题/副标题/金额)
3. 字号普遍偏小,看一眼很累
4. 数字字体太细,缺乏分量感
5. 圆点、小灰链接等元素辨识度低

## 三、技术约束(设计前必读)

### 技术栈
- **React 18 + Tailwind CSS**
- **Tailwind 类**:仅用预定义工具类(无法自定义 tailwind.config 中的扩展色板)。需要自定义颜色就用 CSS 变量。
- **CSS 变量主题系统**:已经搭建了 5 套主题(暖桃黄昏 / 凉雾微光 / 地铁狂欢 / 马德里黄昏 / 夜金)。
  所有组件读 `var(--bn-text-primary)`, `var(--bn-glass)`, `var(--bn-accent)` 等。

### 关键 CSS 变量(都已定义)
```css
--bn-bg                /* 背景渐变 */
--bn-text-primary      /* 主文字 */
--bn-text-secondary    /* 次文字 */
--bn-text-tertiary     /* 三级文字(辅助说明)*/
--bn-accent            /* 强调色 */
--bn-glass             /* 玻璃面板半透明背景 */
--bn-glass-strong      /* 实心一些的玻璃 */
--bn-glass-border      /* 玻璃边框 */
--bn-positive          /* 正向(收入、完成)*/
--bn-negative          /* 负向(支出、错误)*/
--bn-cat-1..4          /* 4 个分类强调色 */
--bn-button-bg / fg    /* 主按钮 */
--bn-row-border        /* 列表分隔线 */
```

### 字体
- 中文:系统默认 sans
- 数字:`.bn-mono` 类,目前用系统等宽字体(用户嫌细)
- **可以引入 Google Fonts**(JetBrains Mono / Space Grotesk / Inter / Manrope 等),只要在 index.html 加 link 标签

### 组件库
- 已有基础组件:`Button`, `GlassPanel`, `Input`(都在 `src/components/ui/`)
- Recharts 用于图表
- shadcn/ui 可引入(还没用)
- 不能用 framer-motion 之外的动画库 — 但 framer-motion 也还没装(可以加)

### 不能做的事
- ❌ 用 localStorage 之外的浏览器存储 API(已经全用 IndexedDB)
- ❌ 引入超大依赖(>200KB gzip)
- ❌ Service Worker(Phase 6 后才考虑)

## 五、待设计的具体内容

### 1. 顶部导航(优先级最高)

**当前问题**:7 个菜单平铺挤在顶部,移动端用汉堡菜单但抽屉很简陋。

**设计要求**:
- 桌面端:顶部水平 nav 条仍然 OK,但要更精致(图标 + 文字?或者只图标 + tooltip?)
- 移动端:**侧拉抽屉**,要有动画,内容不仅是菜单还可以放一些用户身份信息
- 抽屉/菜单要支持**收起手势**(从屏幕右侧滑入,可以拖拽关闭)
- 参考:Notion 的 sidebar、Linear 的命令栏、Bear 笔记的左侧菜单

### 2. 列表视觉(交易记录是最关键代表)

**当前问题**:每行元素没对齐,字号小,层级感弱。

**当前 DOM 结构**:
```
[图标圆 32x32] [主标题 + 副标题] [金额] [删除按钮]
```

**主标题** = 商家(如 "Mercadona")
**副标题** = 详细信息 + 分类 + 日期(如 "在塞维利亚 · 食杂 · 5/3")
**金额** = `−€ 13.45`(支出红/黑,收入绿,带 +/−)

**设计要求**:
- 每个部分**列对齐**(图标列、文字列、金额列固定宽度)
- 主标题字号 ~16px、字重 500
- 副标题字号 ~12-13px、灰度低
- 金额字号 ~16-18px、**等宽数字字体**(tabular-nums)、字重要够
- 行高要够,不能挤
- 划过整行(hover)有 subtle 反馈
- 长按 / 划过时**右侧滑出删除**(iOS 风格)而不是显示小垃圾桶

参考:Apple Wallet 的交易列表、Cash App 的活动列表。

### 3. 字体系统

**当前**:全用 sans-serif 系统字体,数字用系统 mono。
**用户原话**:"字体太小了,数字的字体太丑了,而且太细"

**建议尝试**:
- 数字字体改为 **JetBrains Mono** 或 **Space Mono** 或 **DM Mono**(更有性格,字重可变)
- 中文字体保留系统,但**整体字号 +1-2px**(目前 base size = 14px,提到 15-16px)
- 标题字体可以试 **Inter** / **Manrope** / **Plus Jakarta Sans** 等,带一点几何感

### 4. 番茄钟画中画浮窗

**当前问题**:用户原话 "弹窗的黑框太丑了"。

**当前实现**:浮窗是一个 280x220px 的小窗口,内部用 inline HTML/CSS 写的(用户**很难**看到具体丑在哪里,但反馈说丑就肯定丑)。

**当前样式**:
- 暖桃 / 凉雾渐变背景(根据当前模式)
- 56px 大数字
- 简陋的玻璃按钮
- 字符级别的"暂停 / 继续"按钮

**设计要求**:
- 窗口不大(浏览器 PiP 一般限制在 200-400px 宽),但要"小而精"
- 必要元素:大数字、模式标签(FOCUS / BREAK)、任务名、进度条、暂停 / 停止按钮
- 可以是磨砂玻璃 + 圆角 + 柔和阴影的效果
- 完成时(数字归零)有个庆祝动画 — 至少是颜色变化或一个对勾

### 5. 八段锦 / 早晨问候 / 欢迎页 / 完成动画

**用户原话**:"我 HTML 那些欢迎页、早晨的八段锦、完成的动画都给我删了??"

**老 HTML 的实现**(从原代码扒出来的):
- **启动欢迎页**:fullscreen,中间一行"欢迎回来!",字体超大,渐变色(orange→pink→purple→sky),spring 弹簧动画从下方弹起,gradient 慢慢移动
- **早晨问候**:具体实现已忘,用户记得它存在
- **八段锦**:这是中国传统养生功法 8 个动作,用户暗示"早上提醒做一下"

**设计要求**:
- 启动页:保留(仅在用户登录后第一次进入时显示,或每天首次访问时)
- 时间感知问候:早上"早安"+ 茶杯emoji,中午"午好"+ 太阳,晚上"晚上好"+ 月亮
- 早晨提示卡片(可选):8:00-10:00 之间打开 app 时,首页显示一个小卡片"今天的八段锦?"(用户可以自己点完成或忽略)
- 完成动画:加记账成功、完成番茄钟、完成习惯等场景,需要一个**统一的庆祝动作语言**(可以是 lottie 或 css keyframes)

### 6. 整体视觉气质(灵魂层)

**用户的视觉口味**(从老 HTML 推断):
- 喜欢**有趣的小细节**(渐变文字、弹簧动画、emoji 装饰)
- 喜欢**温暖的颜色**(暖桃、橙、粉)
- 不喜欢"商务风" / "极简到无趣"
- 怀念"地铁狂欢"主题(橙 + 粉 + 紫 + 天蓝四色)的活力

**避坑**:
- ❌ 太冷峻的 SaaS 风(像 Stripe、Linear)
- ❌ 过度极简(像 iA Writer 那种纯白)
- ❌ 太"产品经理范"(规整对称、整齐到无聊)

**靠拢**:
- ✅ 像 Notion,但更有人情味
- ✅ 像 Things 3,但更年轻
- ✅ 像独立设计师做的小工具

## 六、交付物建议

设计师可以按这个顺序交付:

### Phase 1(最高优先级,影响每一页)
1. 顶部导航 / 侧拉抽屉

### Phase 3(锦上添花)
10. 各种空状态插画
11. 加载骨架
12. 数据可视化(图表)的颜色系统
